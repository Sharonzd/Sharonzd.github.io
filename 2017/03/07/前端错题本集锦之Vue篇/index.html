前端错题本集锦之Vue篇

发表于 2017-03-07   |   分类于 前端笔试   |     |   阅读次数

从中学时就养成了用错题本收集错题的习惯, 即便工作了也一直沿用至今, 所以打算写一个系列文章, 将平时前端开发中遇到的坑或者小知识点分享出来, 希望能对大家也有所帮助。知识点杂多, 暂且粗略的分类为HTML篇、JavaScript篇、CSS、Vue篇, 待以后不断扩充之后再做更细致的分类。此篇为其中的Vue篇。

  1. jsonp的时候，会直接改写params的值对象，给它加上callback参数。所以如果需要继续用提供给params的对象，需要新建一个变量。

  2. vue-resource升级：get 参数变为params；post 不变

  3. 子组件里，props数组里的数据确实会自动更新，但是ready只执行一次，并且早于父组件执行，data里的初始化赋值也无法及时获取到，所以不能在ready里用props的数据给其他变量赋值。如果需要使用props，需要用watch来监控。更好的解决办法是，在父组件里data里初始化时就赋予有意义的值。这样，可以减少一次重新渲染，同时子组件也能直接获取到props的值，并且会随着父组件的变化而变化。但是，如果子组件有赋值操作，就需要watch来监控了。

  4. vue的$event能从页面将dom对象传递给js

  5. 在父子组件的通信中。子组件应该保持通用，父组件的事情尽量让父组件自己去做，少用watch

  6. 避免直接修改prop的值，因为值可能会当parent重新被渲染的时候被覆盖。应该用新建一个基于prop的值的data属性，或者新建一个计算属性。

  7. vue2的HTML页面里不能使用命名为in的变量，否则会报错

  8. vue的ajax方法的then 里面的this指向不会变，因为vue-resource已经对其进行了封装

  9. :title 即v-bind:title = 必须要等于一个变量。不能传递空字符串，否则报错。如果要传空字符串。title=””就好了

  10. router-link标签里面不能有a标签，否则点击事件会异常绑定到a标签上
