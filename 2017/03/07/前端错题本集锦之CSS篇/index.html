前端错题本集锦之CSS篇

发表于 2017-03-07   |   分类于 前端笔试   |     |   阅读次数

此篇为前端错题本集锦系列文章中的CSS篇, 分享出来以供大家一起避坑。

  1. 防止动画撑开窗口
    html,body{
    width:100%;
    overflow-x:hidden;/防止动画撑开窗口,注意一定是overflow-x,否则纵向也不能滚动了。此外,虽然overflow是放在width下面的,但是并不影响它会影响width和height这个事实/
    }

  2. 图片左上角固定角标处理方式

解决办法：

1. 横幅放在left:-50%,top:0的位置
2. 文字水平居中垂直居底：父元素为absolute后，table-cell的vertical-align不管用，只能给子元素也absolute，并且left -50%后再通过translate(-50%,-100%)水平移动自身的50%；(先top100%再通过transform: translate(-50%,-100%)垂直移动-100%是因为有的安卓机不能很好的支持bottom属性。）
3. 旋转到左上角：transform-origin修改旋转中心点

代码如下：
HTML:

TOP1

div{
position: absolute;
top: 0;
left: -50%;
width: 100%;
height: .6rem;
transform-origin: 50% 0;
transform: rotate(-45deg);
-webkit-transform: rotate(-45deg);
background-color: #fc2849;
color: #fff;
}
span{
position: absolute;
top: 100%;
left: 50%;
transform: translate(-50%,-100%);
font-size: 12px;
}

  1. 有的android手机不支持bottom，需要用top来替代,如bottom:0可以替换为top:100%;transform:translate(0,-100%)。

  2. 浏览器动态计算宽度。calc() 阻止less错误的计算：calc(~”xxx”)

  3. 自定义select样式。select默认样式无法被覆盖。想要设计select的样式只有以下几种办法。
    a. select的appearance:none ,清除所有样式。
    代码：-webkit-appearance: none;
    b. select的透明度设为0，然后重写一个select覆盖在上面，并用一个span来保存select选择后的内容。
    代码：background: url() no-repeat scroll right center transparent;
    c. 修改select背景图，并把背景图放到最右边的位置。代价是得用个单独的图片，会增加一个http请求
    d. 直接来个小图标，绝对定位定在select框里。 但是，代价是小图标不可点击。
    e. 点击自定义的元素的事件时，如果已知选项的个数，则$().attr(‘size’,7);和$().attr(‘size’,1)切换select元素展开或关闭

  4. -webkit-box没有换行的属性。只有用display:flex;flex-wrap:wrap;才能换行

  5. position:fixed后页面会自动回到顶部。因为fixed元素是脱离文档流的。没有指定top就会top默认为0，回到顶部。所以fixed页面的时候，注意提供top属性

  6. 字体用rem做单位可能回导致计算和缩放时出现锯齿，解决办法是指定dpi。

  7. css匹配从右向左
    http://stackoverflow.com/questions/5797014/why-do-browsers-match-css-selectors-from-right-to-left

  8. 用flex流式布局实现以图片的自适应

  9. 自己实现下边的内阴影：:after{content:””;display:inline-block;position:absolute;width:100%;background-color:#ddd;bottom:0;left:0 ;height:1px;-webkit-transform:scaleY(.5);可以机智得把线处理成0.5px

  10. on状态下拉三角形不需要换样式，只需要旋转180度即可。-webkit-transform:rotate(-180deg);

  11. android的margin-top可能会不起作用，解决办法是加border使得页面强制重新计算元素大小

  12. 单行/多行文字溢出
    a. 单行文本：
    white-space: nowrap;
    width: 100%;
    overflow: hidden; / “overflow” value must be different from “visible” /
    text-overflow: ellipsis;
    b. 多行文本：
    overflow: hidden;
    text-overflow: ellipsis;
    display: -webkit-box;
    -webkit-line-clamp: 3;
    -webkit-box-orient: vertical;

  13. 要给特定的元素的before设置特别的样式，需要先选元素，再选before，而不是从before里选。
    ul li:nth-child(2):before{xxx}

  14. width:100%的同时，设置left，会导致元素超出屏幕

  15. 单位使用rem的话，浏览器会四舍五入，造成错误。所以某些overflow:hidden的地方要注意。

  16. 子元素的box-shadow不生效。可能是因为 父元素overflow:hidden了。
    解决办法是尽量把box-shadow放在父元素上; 或者margin-bottom/margin-top:5px，给子元素留一点发挥空间 ; 或者 移除overflow:hidden

  17. 小圆点实现方式：background: radial-gradient（circle,transparent 10px, #fff 0）

  18. 能写在css中的属性，就写在css中。比如动画：item[0].style.webkitTransition=’margin-top .5s ease-out’可以写在css中：transition:margin-top .5s ease-out;-webkit-transition:margin-top .5s ease-out;
    因为写在css中会从渲染开始就生效。而js要触发才会生效。

  19. 移动端的css的1px不一定为设备的1px。window.devicePixelRatio为物理像素（屏幕上的像素） / 独立像素（css的px）
