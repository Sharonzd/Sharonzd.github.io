前端错题本集锦之HTML篇

发表于 2017-03-07   |   分类于 前端笔试   |     |   阅读次数

前端开发的几年中, 遇到了各种大大小小的坑, 不记录下来便会时常忘记时常再犯, 于是老老实实的无论大小的错误都记录了下来, 现在特地分享出来, 希望能对大家也有所帮助。此篇为HTML篇。

  1. viewport的meta标签，使得width=device-width会基于不同设备宽度进行展现

  2. 页面的性能问题
    a. 加很多class会增加dom树的大小，会降低页面的渲染速度，降低js取元素时遍历树的效率；
    b. 但是不用class，用标签的层级关系查找的话又会增加开发时增加需求时页面结构改变的风险，这一点需要权衡。
    c. 还有页面元素唯一的时候，应该用id，能用id就用id，也可以增加性能；
    d. 但如果是控件就不要用id，因为一个页面可能会有多个

  3. 精简页面无意义元素：一般分割线都用border实现；一般辅助性元素都用before或after


  4. 其中的initial-width=1.0解决iphone、ipad以及IE 会横竖屏不分，通通以竖屏的ideal viewport宽度为准的毛病；
    width=device-width则解决了IE的这个毛病,所以需要两个都加
